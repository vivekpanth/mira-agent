# MIRA — Multimodal Interactive Rehearsal Assistant

> A web app where a student types a scenario, **rehearses a spoken conversation with a 3D
> virtual client** inside a virtual office, and gets an **AI communication-coaching report**
> afterward — with every score backed by a direct quote from the transcript.

Built for the **ACU x AWS Hackathon: Virtual Client Interaction Web App**.
**Team:** Vivek · Narendra · Diwash.

🔒 **Our key design decision: the conversation runs on a *local* AI model — no student or
client data ever leaves the machine.** We use AWS where the cloud adds real value (a natural
neural voice and computer-vision feedback) and keep the sensitive dialogue private and on-device.

---

## ✨ What it does (mapped to the brief)

| Brief requirement | How MIRA delivers it |
|---|---|
| Accept a user-defined scenario via text | Scenario screen: free-text box + example chips |
| Generate a virtual space with a client figure | Procedural 3D avatar in a furnished office (React Three Fiber) |
| Student sees themselves via camera | Live webcam feed on a **monitor inside the 3D scene** — you and the client share one room |
| Speak to the client and practise | Push-to-talk → client replies **aloud in a natural voice (Amazon Polly)** with facial + lighting reactions |
| Client responds naturally | Mood-aware dialogue from a **local LLM**; the avatar and room lighting shift with emotion |
| Record the session + download | Composite recording (3D scene + **burned-in captions** + **both voices**); one-click `.webm` download |
| *(beyond the brief)* Feedback for reflection | A full **coaching report**: overall score, skills radar, measured signals incl. **eye-contact % (Amazon Rekognition)**, evidence quotes, and two concrete tips |

---

## 🔐 Privacy & security (our differentiator)

- **Dialogue stays on-device.** The conversation is generated by a **local LLM — Llama 3.1 8B
  (with Gemma as a fallback) via Ollama.** Nothing about what the student says is sent to any
  third-party LLM API.
- **Speech-to-text is in-browser** (Web Speech API) — the student's audio isn't shipped to a
  cloud transcriber.
- **AWS is used only where it adds value the device can't:**
  - **Amazon Polly** — the client's natural neural voice.
  - **Amazon Rekognition** — objective eye-contact analysis (face pose) for the report.
- This matters for the target setting (health, counselling, professional training), where
  conversation data is sensitive and data-governance rules are strict.

## 🧱 Tech stack

- **Next.js 16** (App Router, Turbopack) + **React 19** + **TypeScript (strict)**
- **Tailwind CSS v4** + **shadcn/ui**
- **3D:** React Three Fiber v9 + drei v10 (procedural primitives — **zero external 3D assets**)
- **Browser APIs:** Web Speech (dictation), Web Audio (mixing for the recording), MediaRecorder + Canvas 2D (composite capture)
- **Local AI:** Ollama — `llama3.1:8b` (fallback `gemma4:e4b`)
- **AWS (live):** Amazon **Polly** (voice) · Amazon **Rekognition** (eye contact)
- **Testing:** Vitest (deterministic metrics tested first)

Every external call sits behind a **Mock / Real / Local switch**, so the app also runs fully
offline on `localhost`.

## 🚀 Quick start

```bash
git clone https://github.com/vivekpanth/mira-agent.git
cd mira-agent
npm install
ollama pull llama3.1:8b      # local dialogue model (private)
npm run dev                  # → http://localhost:3000
```

For the live AWS voice + eye-contact, copy `.env.example` → `.env.local` and add AWS credentials
(see [`docs/AWS_SETUP.md`](docs/AWS_SETUP.md)). The app still runs without them.

## 🗺️ The flow

```
Scenario  ──►  Rehearsal  ──►  Report
 (type)        (speak + 3D       (score + radar + eye-contact %
                virtual client)   + evidence quotes + tips)
```

1. **Scenario** — describe a client (or pick an example). A keyword router builds a matching
   persona, room, mood, and voice.
2. **Rehearsal** — you and the 3D client face each other in an office. Push-to-talk; the client
   replies aloud and reacts. On-screen **speech-bubble captions**; live coaching each turn. The
   session is recorded (scene + captions + both voices).
3. **Report** — an evidence-grounded coaching report: scores, **eye-contact % from Rekognition**,
   talk-ratio/fillers/pace, quotes, and tips. Download the recording or export the report as PDF.

## 📁 Project structure

```
src/
  app/
    page.tsx                 # shell + step router (Scenario→Rehearse→Report)
    api/
      scenario/route.ts      # POST text  -> Persona
      converse/route.ts      # POST turn  -> { replyText, audioUrl, mood }
      analyze/route.ts       # POST turns -> Report  (deterministic metrics + LLM prose)
      vision/route.ts        # POST frame -> { onTarget }  (Amazon Rekognition eye contact)
  components/
    screens/                 # ScenarioScreen, RehearsalScreen, ReportScreen
    ui/                      # PersonaCard, LiveTranscript, MetricChips, SpeechCaption, …
    three/                   # AvatarStage, Avatar, Room  (all R3F, 'use client')
  services/                  # polly, rekognition, LLM router (→ local Ollama) — Mock/Real/Local
  lib/                       # useSession, metrics (deterministic), useCamera, useEyeContact, audioBus
  types.ts                   # FROZEN contracts
```
